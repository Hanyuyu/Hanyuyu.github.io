数据结构回顾 - 栈的底层实现

2019-08-12 Views 栈 数据结构 902字 4 min read 转发:

🙎‍♂️💦之前在博客中, 从0实现了动态数组, 这个数组灵活性相当高, 在容量不够或者容量过大时都会进行resize操作, 且这个动态数组支持泛型, 任何对象都可以成为其元素。
这次，我们依然自己来实现一个栈！

栈在学习过程中出现频率挺高， 举两个例子，第一个是撤销操作，就是我们在文本编辑器中常用到的撤销(Undo)操作，原理和栈是一个道理，还有一个是程序调用的系统栈，在对子过程进行调用的时候，会把主函数中子函数的入口地址存入栈中，待子过程调用完毕后，会从该系统栈中取出主函数中的位置，接着执行剩余的命令。

现在，我们来想一想该如何实现这个栈。🧐

1.需求分析

首先，我们把栈要实现的功能进行罗列：

	void push(E)    //进栈
	E pop()            //出栈
	E peek()          //查看栈顶元素
	int getSize()    //获取栈中元素个数
	boolean isEmpty()   //查看栈是否非空

以上就是我们在栈中要实现的功能，从用户角度来看，设计的栈支持这些操作就好，具体底层的实现，用户并不需要关心，底层实现有多种方式，为满足多态性要求，我们先设置一个Stack接口，定义出基本的操作， 然后我们可以利用以实现的动态数组，来构建自己的ArrayStack类，使其实现Stack接口，这样我们的栈也有了类似动态数组的自动调整容量的resize()方法。

2.具体实现

Stack 接口定义如下

public interface Stack<E> {
    int getSize();
    boolean isEmpty();
    void push(E e);
    E pop();
    E peek();
}

ArrayStack 定义如下

public class ArrayStack<E> implements Stack<E> {

	Array<E> array;
	
	//构造方法用于用户事先知道容量
	public ArrayStack(int capacity) {
		array = new Array<>(capacity);
	}
	//当用户不确定所需容量，提供无参构造函数
	public ArrayStack() {
		array = new Array<>();
	}
	
	
	@Override
	public int getSize() {
		// TODO Auto-generated method stub
		return array.getSize();
	}
	
	@Override
	public boolean isEmpty() {
		// TODO Auto-generated method stub
		return array.isEmpty();
	}

	public int getCapacity() {
		return array.getCapiticy();
	}
	
	@Override
	public E pop() {
		// TODO Auto-generated method stub
		return array.removeLast();
	}

	@Override
	public E peek() {
		// TODO Auto-generated method stub
		return array.getLast(); // 为方便，在Array类中添加了getLast()方法 
	}
	
	@Override
	public void push(E e) {
		// TODO Auto-generated method stub
		array.addlast(e); //时间复杂度复习中, 因为与动态数组一样要考虑到resize操作, 故用均摊法分析复杂度为O(1)
	}
	
	@Override
	public String toString() {
		StringBuilder res = new StringBuilder();
		res.append("Stack: [");
		//res.append(array.toString());
		for(int i=0; i<array.getSize(); i++) {
			res.append(array.get(i));
			if(i != array.getSize()-1)
				res.append(',');
		}
		res.append("] top");
		return res.toString();
	}
	
	
}

这样我们的栈就定义好了。

3.复杂度分析

接着我们需要分析它各项操作的时间复杂度来大致衡量性能
ArrayStack

method 时间复杂度
push O(1) 均摊
pop O(1) 均摊
peek O(1)
getSize O(1)
isEmpty() O(1)

4.验证结果

编写main方法进行验证，如下：

public class Main {
		public static void main(String[] args) {
		ArrayStack<Integer> stack = new ArrayStack<>();
		for(int i=0;i<5;i++) {
			stack.push(i);
			System.out.println(stack);
		}
		stack.pop();
		System.out.println(stack);
		}
}

结果如下
Stack: [0] top
Stack: [0,1] top
Stack: [0,1,2] top
Stack: [0,1,2,3] top
Stack: [0,1,2,3,4] top
Stack: [0,1,2,3] top
